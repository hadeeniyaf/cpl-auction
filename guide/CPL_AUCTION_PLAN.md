## 🎯 AUCTION EXECUTION PLAN

### PRE-AUCTION CHECKLIST (1 hour before)

- [ ] Launch the CPL Auction Tracker app on a laptop/projector
- [ ] Brief all team managers and manager players on rules
- [ ] Confirm each team's budget (projected on screen)
- [ ] Test internet/projector/audio
- [ ] Assign roles:
  - **Auctioneer** — calls players, manages bids
  - **Scorekeeper** — updates live budget on screen
  - **Tech Support** — runs the app, handles exports

### AUCTION ORDER (Category-based)

Run players in this order to build excitement:

1. **🌟 Icon Players (12)** — Opens bidding wars, sets tone
2. **🥅 Goalkeepers (5)** — High demand, each team needs one
3. **🛡️ Defenders (27)** — Bulk of squad, varied prices
4. **⚙️ Midfielders (16)** — Creative bids, mid-tier value
5. **⚡ Forwards (14)** — Attacking options, exciting bids
6. **🌱 Young Players (~15)** — Ends auction, budget cleanup

### BIDDING MECHANICS

**For each player:**
1. Auctioneer displays player on screen (name, age, position, category, base price)
2. Auctioneer calls out: *"We have [Player Name], [Category], base price ₹[X]. Do we have ₹[X]?"*
3. Team managers bid by raising hand or saying the amount
4. Auctioneer counts down: *"Going once… going twice… going three times… SOLD to [Team Name] for ₹[X]!"*
5. App is updated: team budget auto-deducts, squad is updated
6. Proceed to next player

**Bid Rules:**
- Minimum opening bid = Base price
- Minimum increment = ₹50
- No team can go below ₹200 reserve (emergency fund)
- If no bid within 10 seconds → player marked UNSOLD
- Unsold players can be re-auctioned at end of auction at base price

## 📱 USING THE AUCTION TRACKER APP

### Three Main Views

#### 1. **⚡ AUCTION** (Main Control Panel)
- **Left side:** Current player up for auction
  - Player name, age, position, category, base price
  - Input field for bid amount (auto-fills with base price)
  - Dropdown to select winning team
  - Buttons: **SOLD** | **UNSOLD** | **SKIP →**
- **Right side:** Live team budgets + recent sales log

**Workflow:**
1. Auctioneer announces player
2. Takes bids from teams
3. Enters final bid amount
4. Selects winning team from dropdown
5. Clicks **SOLD** button
6. App auto-updates budget & squad
7. Moves to next player automatically

#### 2. **🛡️ SQUADS** (Real-time Team Rosters)
View each team's full squad:
- Manager player (pre-assigned)
- All purchased players with prices
- Total spent vs budget remaining
- Squad size count

#### 3. **📋 ALL PLAYERS** (Searchable Registry)
- Filter by category (Icon, Forward, Midfielder, etc.)
- Search by player name
- See sold/unsold status
- View final prices

### Key Features

✅ **Auto-save to Browser:** All data saved locally — never lose progress  
✅ **Export to CSV:** Click **📥 EXPORT CSV** after auction ends  
✅ **Live Budget Tracking:** Teams' remaining budget updates in real-time  
✅ **Reset Option:** Start fresh with **🔄 RESET** (clears all data)

## 🚨 COMMON SCENARIOS & SOLUTIONS

### Scenario 1: Team Runs Out of Budget
**Solution:** Team cannot bid further. They're locked out of selection until auction ends.

### Scenario 2: A Player Receives No Bids
**Solution:** Mark as **UNSOLD**. Can re-auction at base price at the end if desired.

### Scenario 3: Team Wants to Go Below ₹200 Minimum
**Solution:** Not allowed. Enforce the ₹200 reserve to ensure all teams can still bid for young players.

### Scenario 4: Accidental Wrong Bid Entry
**Solution:** Click **SKIP →**, press back, or use **🔄 RESET** to start over (if early). Data is saved locally.

### Scenario 5: Manager Disputes Final Price
**Solution:** Auctioneer has final say. Once "SOLD" is clicked, transaction is locked.
